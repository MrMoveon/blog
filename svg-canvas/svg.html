        # svg viewport 和 viewBox

        • viewport 是 svg 图像的可见区域
        • viewBox 是用于在画布上绘制 svg 图形的坐标系统
        <svg viewBox="0 0 100 100" style="border: 1px solid #000000">
	<rect x="10" y="45" width="80" height="10" fill="red" />
	<rect x="45" y="10" width="10" height="80" fill="red" />
</svg>

        • 上述案例中绘制了一个按照viewBox（坐标系100*100）的+图标。

        • 由于图标是按照viewBox进行绘制的，所以我们可以设置svg的宽度高度，进行图标的放大缩小：

        <svg width="50" height="50" viewBox="0 0 100 100" style="border: 1px solid #000000">
	<rect x="10" y="45" width="80" height="10" fill="red" />
	<rect x="45" y="10" width="10" height="80" fill="red" />
</svg>

        • 也可以取消svg宽高,元素的fill颜色设置currentColor，给svg父级设置宽高颜色，进行自适应
        <div style="width:200px;height:200px;color:green">
    <svg viewBox="0 0 100 100" style="border: 1px solid #000000">
        <rect x="10" y="45" width="80" height="10" fill="currentColor" />
        <rect x="45" y="10" width="10" height="80" fill="currentColor" />
    </svg>
</div>

        # svg 组件

        在svg中关于多个图形的复用，是通过标签defs来解决的

        解决方案

        defs： 被引用元素的容器 – 这很好理解相当于vue中的component

        g： group缩写 无实意。相关元素整合的容器 < id作为该的唯一标识 >

        symbol： symbol 与 g 标签类似，但symbol可以拥有一个独立的 viewBox （图标库一般使用symbol）

        <svg width="0" height="0" aria-hidden="true">
    <defs>
        <g id="icon-add">
            <rect x="10" y="45" width="80" height="10" fill="red" />
            <rect x="45" y="10" width="10" height="80" fill="red" />
        </g>
        <!-- 使用symbol，拥有独立的坐标系-->
        <symbol id="icon-menu" viewBox="0 0 100 100">
            <rect x="10" y="10" width="80" height="10" fill="red" />
            <rect x="10" y="40" width="80" height="10" fill="red" />
            <rect x="10" y="70" width="80" height="10" fill="red" />
        </symbol>
    </defs>
</svg>

<!-- 使用icon-add图标，设置宽高，图标不会跟着放大缩小 -->
<svg width="200" height="200">
    <use href="#icon-add" />
</svg>

<!-- 使用icon-menu图标，设置宽高图标会跟着放大缩小 -->
<svg width="200" height="200">
    <use href="#icon-menu" />
</svg>

        以上代码在第一个svg中，设置了宽高为0，aria-hidden="true"让这个元素对浏览器进行语义化的隐藏，避免设备的抓取。

        svg中定义了2个图标，采用了g和symbol

        g：icon-add图标，设置宽高，图标不会跟着放大缩小。

        symbol：icon-menu图标，设置宽高，图标会跟着放大缩小

        const path = require("path");
// 获取绝对路径
function resolve(name) {
  return path.resolve(__dirname, name);
}
module.exports = {
  chainWebpack(config) {
    //默认的svg 规则 排除icons目录
    config.module
      .rule('svg')
      .exclude.add(resolve('src/icons/svg'))
      .end()
    // 新增icons规则，只包含icons目录
    config.module
      .rule('icons')
      .test(/\.svg$/)
      .include.add(resolve('src/icons/svg'))
      .end()
      .use('svg-sprite-loader')
      .loader('svg-sprite-loader')
      .options({ symbolId: 'icon-[name]' }) //使用图标名
      .end()
  }
};

        使用`require.context函数遍历svg文件夹的所有文件,一次性导入到index.js中。

        /**
 * @description 自动加载svg目录下的.svg文件
 * 语法: require.context(directory, useSubdirectories = false, regExp = /^.//);
 * require.context函数接受三个参数
 * directory {String} -读取文件的路径
 * useSubdirectories {Boolean} -是否遍历文件的子目录
 * regExp {RegExp} -匹配文件的正则
 */
const req = require.context("./svg", false, /\.svg$/);
req.keys().map(req);

// 值得注意的是require.context函数执行后返回的是一个函数,并且这个函数有3个属性
// resolve {Function} -接受一个参数request,request为test文件夹下面匹配文件的相对路径,返回这个匹配文件相对于整个工程的相对路径
// keys {Function} -返回匹配成功模块的名字组成的数组
// id {String} -执行环境的id,返回的是一个字符串,主要用在module.hot.accept,应该是热加载?

        # 封装svg组件

        props传递2个参数，一个图标名字type,一个class类名class-name

        aria-hidden="true" 让这个元素对浏览器进行语义化的隐藏，避免设备的抓取

        v-bind="$attrs" 将父组件的属性（除去在props中传入的属性）传递给子组件

        <template>
  <svg :class="svgClass" v-bind="$attrs" aria-hidden="true">
    <use :xlink:href="iconName" />
  </svg>
</template>

<script>
export default {
  props: {
    type: {
      type: String,
      required: true
    },
    className: String
  },
  computed: {
    iconName() {
      return `#icon-${this.type}`
    },
    svgClass() {
      if (this.className) {
        return `svg-icon ${this.className}`
      }
      return `svg-icon`
    }
  }
}
</script>

<style lang="scss">
.svg-icon {
  display: inline-block;
  font-size: 1em;
  width: 1em;
  height: 1em;
  fill: currentColor;
  overflow: hidden;
}
</style>

        # svg+css3 实现tabbar 动效

        大家好，今天给大家分享下svg如何实现一个简单的tabbar动效，并适配移动端平台。

        # 动效分析

        我们先来分析一下如图tabbar的元素动一动的过程：

        • 动画开始阶段：整个图标进行类似心跳效果的抖动，并且内部小图标进行消失
        • 动画运行到中间：图标内部有三层颜色的闪屏效果，运动结束并停止到最后一层
        • 动画最后阶段：尬聊图标中间小圆跳动显示，其他三个内部小图标变成白色并进行线条移动显示。

        # tabbar 图标绘制

        动画分析后，我们先来进行准备工作，由于这个动效没有原始文件，所以我们还需自己在软件上进行48x48大小的图标绘制导出，或者自己用代码就行绘制。

        一般设计师都会提供源文件，所以我就不进行图标绘制的教程了。

        # 中间小圆路径消失动画

        路径动画关键在于属性stroke-dasharray 和 stroke-offset。

        1. stroke-dasharray： 用于创建虚线 ；
        2. stroke-dashoffset：整个路径的偏移值；

        我们先通过js获取小圆的路径长度，约为44：

        let homeCirclePath = document.querySelector('#home-circle')
console.log(homeCirclePath.getTotalLength()); // 43.9885 约等44

        在scss中设置圆的虚线长度和偏移值，并定义关键帧，让圆的路径偏移44达到路径消失的效果，这时我们就可以看到小圆的路径动画了

        
.tabbar{
	#home-circle{
	    stroke-dasharray:44 44;
	    stroke-dashoffset:0;
	}
	.on{
		#home-circle{
			animation: home-circle 0.5s ease forwards;
		}
	}
}

@keyframes home-circle{
    0%{stroke-dashoffset:0;}
    100%{
        stroke-dashoffset:44;
    }
}

        # keyframs的@minxi

        首页、探索、我的三个图标内部的图形都是路径动画,并且效果一致，我们可以封装一个@minxi，直接调用:

        /**
 * @params {string} $name 关键帧名字
 * @params {string} $length 路径长度
 * @params {string} $startColor 开始动画路径的颜色
 * @params {string} $endColor 结束动画路径的颜色
 */
@mixin pathKeyframes($name,$length,$startColor:#B1BACD,$endColor:#ffffff) {
	@keyframes #{$name}{
		0%{
			stroke:$startColor;
			stroke-dashoffset:0;
			}
		50%{
			stroke:$startColor;
			stroke-dashoffset:$length;
		}
		50.1%{
			stroke:$endColor;
			stroke-dashoffset:$length*-1;
		}
		100%{
			stroke:$endColor;
			stroke-dashoffset:0;
		}
	}
}
// 生成首页图标的小圆路径动画
@include pathKeyframes(home-circle,44);
// 生成探索图标的箭头路径动画
@include pathKeyframes(explore-arrow,39);
// 生成我的图标的中间线路径动画
@include pathKeyframes(me-line,23);

        给图标加上虚线和偏移值默认值为0，再给选中状态下，添加animation动画。
