    # 抽象类、抽象方法

    typescript 中的抽象类，是其他子类继承的基类，不能被实例化

    用 abstract 定义抽象类、抽象方法，抽象类中的抽象方法需要在子类去实现

    抽象类和抽象方法是用来定义标准，子类必须按照标准去实现

    
abstract class People {
    abstract eat():any  //抽象类中的抽象方法需要在子类去实现
}

class Man extends People{
    eat():void{
        console.log('吃饭');

    }
}

    # 可选属性

    接口里的属性在某些条件下存在，或者不存在

    interface Cricle {
  color?: string;
  r: number;
}
function getCricle(attr: Cricle): void {
  if (attr.color) {
    console.log(`${attr.color}的圆`);
  }
  console.log("面积：" + Math.PI * attr.r * attr.r);
}
getCricle({ color: "白色", r: 10 }); //白色的圆,面积：314.1592653589793
getCricle({ r: 20 }); // 面积：1256.6370614359173

    # ts 封装 ajax

    interface Config {
  type: string; //;结尾
  url: string;
  data?: string; //?可选参数
  dataType: string;
}

function ajax(config: Config) {
  let xhr = new XMLHttpRequest();
  xhr.open(config.type, config.url, true);
  xhr.send(config.data);
  xhr.onreadystatechange = function() {
    if (xhr.status === 200 && xhr.readyState === 4) {
      console.log("成功");
      if (config.dataType === "json") {
        console.log(JSON.parse(xhr.responseText));
      } else {
        console.log(xhr.responseText);
      }
    }
  };
}

ajax({
  type: "GET",
  url: "https://cnodejs.org/api/v1/topics?page=1&limit=2",
  data: "",
  dataType: "json"
});

    # 函数类型接口

    对方法传入的参数和返回的结果进行约束

    // 加密接口
interface encrypt {
  (key: string, value: string): string;
}

var md5: encrypt = (key: string, value: string): string => {
  return key + value;
};
var sha1: encrypt = (key: string, value: string): string => {
  return key + "--" + value;
};
console.log(md5("name", "lisi")); // namelisi

console.log(sha1("name", "lisi")); // name--lisi

    # 接口扩展

    我们实现一个复杂的示例:

    Workers 接口继承 People 接口,WebEngineer 继承 Programmer 并实现 Workers 接口

    //  人
interface People {
  eat(): void;
}
//   工作者
interface Workers extends People {
  work(): void;
}
//   程序员
class Programmer {
  name: string;
  constructor(name: string) {
    this.name = name;
  }
  code(): void {
    console.log(this.name + "写代码啦~");
  }
}
//   工程师
class WebEngineer extends Programmer implements Workers {
  constructor(name: string) {
    super(name);
  }

  eat(): void {
    console.log(this.name + "需要吃饭呀~~~");
  }
  work(): void {
    console.log(this.name + "需要工作~~");
  }
}

var w = new WebEngineer("小李");

w.eat(); // 小李需要吃饭呀~~~
w.code(); // 小李写代码啦~
w.work(); // 小李需要工作~~

    # 泛型

    通俗地讲：泛型就是解决类、接口、方法的复用性，以及对不特定数据类型的支持。

    # hello word

    下面来创建第一个泛型的示例，getData 函数，这个函数会返回任意传入的值。 不用泛型的话可能是这样的：

    function getData(value: string): string {
  return value;
}

    或者用 any 类型定义函数：

    function getData(value: any): any {
  return value;
}

    使用 any 类型会导致这个函数可以接收任何类型的 value 参数，这样就不能对数据类型进行校验。

    getData 泛型的使用方式

    function getData<T>(value: T): T {
  return value;
}
var result1: string = getData<string>("hello world")
console.log(result1); // hello world
var result2: number = getData<number>(1314520)
console.log(result2); // 1314520

    # 泛型类

    通过泛型类计算出最小值

    class Min<T>{
    public list: T[] = [];
    constructor(list: T[]) {
        this.list = list
    }
    getMin(): T {
        let min = this.list[0];
        for (let i = 0; i < this.list.length; i++) {
            if (min > this.list[i]) {
                min = this.list[i]
            }
        }
        return min
    }
}

let m1 = new Min<number>([10,4,6,8,54]);
console.log(m1.getMin()); // => 4

let m2 = new Min<string>(['y','h','g','a']);
console.log(m2.getMin()); // => a

    把类作为参数来约束传入的类型

    以封装操作mysql,mssql的底层类库为示例，进行详细讲解展示：

    要求：mysql,mssql功能一样，都有add update delete find方法

    注意：约束同意的规范，以及代码重用

    解决方案：需要约束规范所以要定义接口，需要重用所以需要用到泛型

    1/接口：在面向对象的编程中，接口是一种规范的定义，它定义了行为和动作的规范

    2/泛型：就是解决类 接口 方法的复用

    interface IDB<T>{
    add(info:T):boolean;
    update(info:T,id:number):boolean;
    delete(id:number):boolean;
    find(id:number):any[];
}

// 定义MySql类 并实现IDB接口，
// 如果没有实现接口方法，vscode可以点击 MySql下面的波浪下，点击自动修复，会自动实现接口的方法
class MySql<T> implements IDB<T>{
    add(info: T): boolean {
        console.log('MySql add:');
        console.log(info);
        return true
    }    
    update(info: T, id: number): boolean {
        throw new Error("Method not implemented.");
    }
    delete(id: number): boolean {
        throw new Error("Method not implemented.");
    }
    find(id: number): any[] {

        let list =[
            {title:"文章1",desc:"文章描述"},
            {title:"文章2",desc:"文章描述"}
        ]
        console.log('find list:');
        console.log(list);
        
        return list
    }
}
// 定义MsSql类 并实现IDB接口
class MsSql<T> implements IDB<T>{
    add(info: T): boolean {
        console.log('MsSql add:');

    }

    }

            {title:"文章2",desc:"文章描述"}
        ]
        console.log('find list:');

        return list
    }
}

    # 命名空间

    “内部模块”现在称做“命名空间”。是为了避免多人协作产生代码冲突。

    把你的代码放到 命名空间 namespace里面

    要访问命名空间的方法、类、变量，需要把方法、类、变量进行导出

    一个模块可以存在多个命名空间，命名空间也支持export进行导出

    // moduleA.js
