    # 基本配置

    # 添加脚本

    在 package.json 里加一些脚本

    {
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}

    本地开发使用如下命令，它会在本地创建一个服务器，你可以使用 localhost:8080 进行访问

    npm docs:dev

    要生成静态的 HTML 文件，运行以下命令，它会在.vuepress 下生成一个 dist 目录

    npm run docs:build

    默认情况下，文件将会被生成在 .vuepress/dist，当然，你也可以通过 .vuepress/config.js 中的 dest 字段来修改，生成的文件可以部署到任意的静态文件服务器上

    # 项目目录

    进行文档编写，需要遵循 vuepress 的目录结构

    |-- docs // 特定的目录
    |-- README.md // 首页
    |-- .vuepress // 特定的目录
        |-- config.js // 特定的配置文件
    |-- electron  // 文档文件夹
        |--README.md // 文档主页
    |-- typescript  // 文档文件夹
        |--README.md // 文档主页
|-- package.json // 脚本命令

    # 配置 ico 图标

    配置 ico 图标需要在 config.js 进行设置，icon.ico 需要放到.vuepress/public 文件夹下面，他的配置如下：

    module.exports = {
  head: [["link", { rel: "icon", href: "/icon.ico" }]]
};

    # 导航栏

    # 基础配置

    配置顶部导航栏，需要在 config.js 的 themeConfig 进行配置，跳转到第三方链接，直接在 lin'k 里配置网址，配置如下：

    module.exports = {
  themeConfig: {
    // 顶部导航配置
    nav: [
      { text: "主页", link: "/" },
      {
        text: "electron",
        link: "/electron/"
      },
      {
        text: "vuepress",
        link: "/vuepress/"
      },
      { text: "github", link: "https://google.com" }
    ]
  }
};

    # 下拉列表

    下拉列表需要在导航里添加 items 数组进行显示,还可以通过嵌套的 items 来在 下拉列表 中设置分组

    module.exports = {
  themeConfig: {
    // 顶部导航配置
    nav: [
      { text: "主页", link: "/" },
      {
        text: "electron",
        items: [
          {
            text: "electron 基础",
            link: "/electron/base"
          },
          {
            text: "electron 进阶",
            link: "/electron/advance"
          }
        ]
      },
      //还可以通过嵌套的 items 来在 下拉列表 中设置分组
      {
        text: "Languages",
        items: [
          {
            text: "Group1",
            items: [
              /*  */
            ]
          },
          {
            text: "Group2",
            items: [
              /*  */
            ]
          }
        ]
      },
      { text: "github", link: "https://google.com" }
    ]
  }
};

    # 禁用导航栏

    可以使用 themeConfig.navbar 来禁用所有页面的导航栏

    module.exports = {
  themeConfig: {
    navbar: false
  }
};

    或者在某个页面中禁用导航栏

    ---
navbar: false
---

    # 自动生成侧边栏

    如果你希望自动生成一个仅仅包含了当前页面标题（headers）链接的侧边栏，你可以通过以下配置来实现：

    module.exports = {
  themeConfig: {
    sidebar: "auto"
  }
};

    你也可以通过 YAML front matter 来实现

    ---
sidebar: "auto"
---

    # 使用vue组件

    所有在 .vuepress/components 中找到的 *.vue 文件将会自动地被注册为全局的异步组件,例如：

    ├─.vuepress
│  │  config.js
│  │
│  ├─components
│  │      ContextMenu.vue
│  │      ContextMenuDemo.vue

    你可以直接使用这些组件在任意的 Markdown 文件中（组件名是通过文件名取到的）：

    <context-menu-demo></context-menu-demo>

    # 使用预处理器

    VuePress 对以下预处理器已经内置相关的 webpack 配置：sass、scss、less、stylus 和 pug。要使用它们你只需要在项目中安装对应的依赖即可。例如，要使用 less，需要安装：

    npm install less less-load -D

    组件中使用

    <style lang='less'>
.context-menu-demo{
  .menu{
      display: flex;
      justify-content: center;
  }
  .el{
      width: 100px;
      height: 100px;
      margin: 10px;
      background: skyblue
  }
  .el:nth-child(2n){
      background: peru;
  }
}
</style>

    # 自动部署

    文档写好后，需要部署到服务器，这里以部署到 github 为例

    github 创建项目，并创建分支 gh-pages

    根目录创建 deploy.sh,命令行执行 deploy.sh,以下命令会自动执行，部署到 gitHub 进行展示

    #!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
# git push -f git@github.com:<USERNAME>/<USERNAME>.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
git push -f https://github.com/MrMoveon/vblog.git master:gh-pages

cd -
